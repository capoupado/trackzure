How to create a Personal Access Token
  1. Open your Azure DevOps organization in a browser.
  2. Click your avatar in the top-right corner → Personal access tokens.
  3. Click New Token.
  4. Give it a name (e.g. "Trackzure"), set an expiry, and grant at least Work Items — Read & write scope.
  5. Copy the token immediately — it won't be shown again.
Self-signed or internal certificate issues

If your Azure DevOps Server uses a self-signed or internal CA certificate, network requests from the extension may fail with a connection error. To resolve this, open your server URL in the browser and accept / trust the certificate there first. The extension uses the same browser trust store, so once trusted in the browser it will work here too.

On-premises collection URL format

Azure DevOps Server (TFS) organises projects inside collections. The Server URL must include the collection segment, for example:

https://tfs.company.com/DefaultCollection

The Project Name field is the short project name that appears after the collection in the URL — e.g. if your work items live at https://tfs.company.com/DefaultCollection/MyProject/_workitems, enter MyProject.
